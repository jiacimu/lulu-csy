# 生图功能用户手册（草稿）

更新时间：2026-05-28

这份手册面向普通使用者，说明如何配置、如何在聊天里手动生图、如何让角色主动发照片，以及遇到问题时怎么排查。本文先按当前界面和功能写成草稿，后续可以继续改成正式版、图文版或内置 FAQ。

## 1. 功能简介

生图功能用于让角色在聊天中发送图片。当前支持两种使用方式：

| 使用方式 | 适合场景 | 主要特点 |
| --- | --- | --- |
| 手动生图 | 你明确想生成一张图 | 在聊天面板点击“生图”，填写画面要求后生成并发送 |
| 角色主动发照片 | 让角色在合适的聊天时机自然发图 | 开启后，角色可以根据对话判断是否要发照片 |

生图结果会同时出现在聊天窗口和对应角色的相册中。点击聊天里的图片可以预览原图，也可以保存图片。

## 2. 使用前准备

### 2.1 配置生图服务

进入：

```text
系统设置 -> 生图服务
```

选择一个生图供应商：

| 供应商 | 需要填写 | 适合谁 |
| --- | --- | --- |
| NovelAI | API URL、Persistent API Token、模型、尺寸、Steps、Scale、Sampler、Schedule、全局质量 tags、全局负面 tags | 熟悉 NAI tags，想控制 tag、Vibe、模型参数的用户 |
| OpenAI 兼容 | Base URL、API Key、模型名、Size、Response、全局自然语言画质、全局避免内容 | 想用自然语言描述画面的用户，或使用兼容 `/images/generations` 的生图接口 |

配置完成后点击：

```text
保存生图配置
```

如果你有多套生图接口，可以在“API 配置预设”里保存当前配置。之后可以一键切换，不用重复填写 key 和模型。

### 2.2 配置风格预设

在“生图服务”页面下方可以维护“风格预设库”。

| 预设类型 | 写法 | 生效范围 |
| --- | --- | --- |
| NovelAI Tag 预设 | 正向 tags、负向 tags 分开写 | 只用于 NovelAI |
| OpenAI 兼容自然语言风格 | 用摄影、插画、氛围、画质等自然语言描述 | 只用于 OpenAI 兼容接口 |

风格预设可以复制、删除，也可以新增。旧版通用预设会显示在“旧版通用预设”中，建议把内容复制出来，分别保存成 NovelAI 或 OpenAI 兼容风格。

### 2.3 为角色开启生图入口

进入某个角色的聊天窗口，打开聊天设置：

```text
聊天 -> 加号面板 -> 设置 -> 生图 / 发照片
```

这里有两个开关：

| 开关 | 作用 |
| --- | --- |
| 手动生图入口 | 打开后，加号面板里会出现“生图”按钮 |
| 角色主动发照片 | 打开后，角色可以在聊天中主动尝试发送照片 |

如果你看不到“生图”按钮，优先检查“手动生图入口”是否已开启。

### 2.4 设置锁脸信息

锁脸用于让生成结果更稳定地接近角色或用户本人。

在同一个“生图 / 发照片”设置里可以填写：

| 项目 | NovelAI 写法 | OpenAI 兼容写法 |
| --- | --- | --- |
| 角色外貌 | 英文 NAI tags，例如 `1boy, short black hair, dark eyes` | 自然语言，例如“黑色短发，深色眼睛，常穿深色衬衫，气质冷淡” |
| 我的外貌 | 合照用 NAI tags，建议不要写 `solo` | 自然语言描述自己的外貌 |

也可以在：

```text
个人档案 -> 我的生图锁脸
```

维护自己的锁脸信息。自动发图时，只有在合照、双人、约会等语境中，才会自动带上“我的锁脸”。手动生图时，可以在弹窗里手动打开“使用我的锁脸”。

## 3. 手动生图

### 3.1 打开入口

进入角色聊天窗口：

```text
点击加号 -> 生图
```

如果按钮没有出现，请先到聊天设置里打开“手动生图入口”。

### 3.2 选择模式

手动生图有两种模式：

| 模式 | 是否调用副 API | 适合场景 |
| --- | --- | --- |
| 手写模式 | 不调用副 API | 你已经知道要写什么 prompt，想直接控制画面 |
| 剧情模式 | 会调用副 API | 想让系统结合最近聊天、角色设定和你的要求，自动整理成更适合生图的画面 |

手写模式会直接使用你输入的内容：

- 使用 NovelAI 时，建议直接写英文 tags。
- 使用 OpenAI 兼容接口时，可以写自然语言画面描述。

剧情模式会读取最近聊天和角色设定，再由副 API 整理成生图画面。使用剧情模式前，需要先配置副 API。

### 3.3 填写画面要求

在输入框中写你想要的画面。可以写得简短，也可以写得具体。

示例：

```text
雨夜车站，角色回头看镜头，外套被雨打湿，路灯反光，情绪克制但很想靠近。
```

NovelAI 手写模式示例：

```text
1boy, solo, looking at viewer, wet hair, black coat, rainy night, train station, cinematic lighting, close-up
```

OpenAI 兼容手写模式示例：

```text
一张雨夜车站的近景照片，角色穿黑色外套回头看向镜头，头发微湿，路灯在地面积水中反光，画面有电影感和克制的暧昧气氛。
```

### 3.4 选择风格和锁脸

生成前可以调整：

| 选项 | 建议 |
| --- | --- |
| 使用角色锁脸 | 单人照、角色主角图建议打开 |
| 使用我的锁脸 | 合照、约会、牵手、一起自拍时再打开 |
| 风格预设 | 选择适合当前供应商的风格；不想套风格时选“不使用风格预设” |
| 参考图（Vibe） | 仅 NovelAI 可用，用于靠近参考图的画风、色调或氛围 |

点击：

```text
生成并发送
```

生成成功后，图片会作为角色消息发送到聊天里，并保存到该角色相册。

## 4. Vibe 参考图

Vibe 目前只支持 NovelAI。

在手动生图弹窗中，可以上传参考图或选择“我的 Vibe”。支持：

- PNG
- JPG / JPEG
- WEBP
- `.naiv4vibe`

暂不支持 `.naiv4vibeBundle`。

每次生图最多选择 3 张 Vibe 参考图。

| 参数 | 含义 | 建议 |
| --- | --- | --- |
| 参考强度 | 参考图对结果的影响强弱 | 想轻微借鉴选“低”，想明显靠近选“高” |
| 提取程度 | 从参考图中提取多少信息 | 默认即可；想更强烈继承风格时选“强烈” |

上传普通图片后，可以点保存按钮加入“我的 Vibe”。已保存的 Vibe 可以重命名、删除、清除编码缓存。

注意：Vibe 只支持部分 NovelAI v4 / v4.5 模型。如果当前模型不支持，生成时会提示“不支持 Vibe 参考图”。

## 5. 角色主动发照片

开启路径：

```text
聊天 -> 加号面板 -> 设置 -> 生图 / 发照片 -> 角色主动发照片
```

角色主动发照片需要：

- 生图服务已经配置完整。
- 副 API 已配置。
- 当前角色已开启“角色主动发照片”。

开启后，角色不会每轮都发图。系统会根据聊天内容判断是否适合发照片，例如：

```text
来张自拍。
```

```text
你现在那里是什么样子？拍给我看看。
```

```text
想看你刚才说的那个房间。
```

如果判断通过，会先出现“发送图片中...”的占位消息，生成完成后替换为图片。主动发照片会优先使用该角色绑定的默认风格和默认 Vibe。

如果照片生成失败，系统会避免让角色继续假装已经发过照片；你可以重新提出生图要求，或改用手动生图。

## 6. 角色风格和默认 Vibe

在角色聊天设置的“生图 / 发照片”里，可以配置：

| 设置 | 作用 |
| --- | --- |
| 角色风格预设 | 勾选该角色可用的风格；点击某个风格可设为默认 |
| 角色默认 Vibe | NovelAI 下可选最多 3 个；主动发照片时会自动使用 |

手动生图时，如果本次没有额外选择 Vibe，会自动使用角色默认 Vibe。

## 7. 相册与图片查看

生成成功后，图片会保存到：

```text
相册 -> 对应角色相册
```

在相册中可以：

- 按角色查看所有照片。
- 点击照片查看大图。
- 删除单张照片。
- 让角色点评照片。
- 查看照片生成时附近的聊天上下文。

在聊天中点击图片，可以打开原图预览，并使用预览页的保存按钮保存图片。

## 8. 调用与消耗说明

不同模式的调用大致如下：

| 操作 | 可能调用 |
| --- | --- |
| 手写模式手动生图 | 生图接口 1 次 |
| 剧情模式手动生图 | 副 API 1 次 + 生图接口 1 次 |
| 角色主动发照片 | 当前聊天回复之外，额外副 API 1 次 + 生图接口 1 次 |
| NovelAI Vibe 首次使用 | 每张 Vibe 可能先编码 1 次，再生图 1 次 |
| 相册中“让 TA 点评照片” | 主 API 1 次，且需要模型支持图片识别 |

失败重试、重新生成、切换模式再次生成，都会增加调用次数。

## 9. 常见问题

### 9.1 看不到“生图”按钮

检查：

1. 是否进入了具体角色的聊天窗口。
2. 是否打开“聊天设置 -> 生图 / 发照片 -> 手动生图入口”。
3. 是否在聊天加号面板中查看。

### 9.2 “生成并发送”按钮不可用

通常是当前生图供应商没有配置完整。进入：

```text
系统设置 -> 生图服务
```

检查 token、key、模型名、Base URL 等配置，并保存。

### 9.3 剧情模式提示需要副 API

剧情模式需要副 API 整理画面。如果没有配置副 API，可以先切回“手写模式”。

### 9.4 NovelAI 结果不像角色

可以尝试：

- 完善“角色外貌 NAI tags”。
- 把稳定外貌写到锁脸里，而不是每次都写在 prompt 里。
- 使用更合适的角色默认风格。
- 减少互相冲突的 tags。
- 合照时再打开“使用我的锁脸”，单人照不要带用户 tags。

### 9.5 OpenAI 兼容接口没有返回图片

检查：

- Base URL 是否是兼容 `/images/generations` 的接口地址。
- 模型名是否正确。
- API Key 是否有效。
- Response 格式是否适合该接口。
- 接口是否只返回了文字说明，而不是图片 URL 或 base64 图片。

可以先点击“拉取模型”确认接口能连通。

### 9.6 Vibe 上传后不能用

检查：

- 当前供应商是否为 NovelAI。
- 当前 NovelAI 模型是否支持 Vibe。
- 文件是否为 PNG、JPG、WEBP 或 `.naiv4vibe`。
- 是否超过 3 张参考图。

### 9.7 角色不主动发照片

检查：

- 是否开启了“角色主动发照片”。
- 生图服务和副 API 是否都配置完整。
- 当前对话是否有明确发图需求。
- 最近是否已经有类似照片，系统可能会避免重复。

可以直接说“来张自拍”“拍给我看”“想看你那边现在的样子”，触发会更稳定。

## 10. 写 prompt 的小技巧

### NovelAI

推荐按这个顺序写：

```text
主体, 外貌, 表情, 姿势, 服装, 场景, 镜头, 光线, 氛围
```

示例：

```text
1boy, solo, black hair, dark eyes, gentle smile, looking at viewer, black shirt, bedroom, night, close-up, warm indoor lighting
```

负面 tags 可以写：

```text
lowres, blurry, bad hands, extra fingers, watermark, text
```

### OpenAI 兼容

推荐写成完整画面：

```text
一张近距离手机自拍，角色坐在暖黄灯光的卧室里，穿黑色衬衫，微微低头看镜头，表情克制但温柔。画面像真实聊天中刚拍下来的照片，构图自然，光线柔和，避免水印、文字和夸张滤镜。
```

### 合照

合照时建议明确写：

```text
双人同框、两个人都清晰、自然靠近、不是拼贴、没有第三人。
```

同时打开“使用我的锁脸”，并选择适合合照的风格预设。
