# 模型调用次数说明（用户版）

更新时间: 2026-05-19

这份只说明“会不会调用主模型 / 副模型，以及大概调用几次”。
不讲接口地址，不讲技术细节，方便直接发给用户看。

## 先说明

- 主模型: 设置里配置的主要聊天模型，负责用户能直接看到的回复、剧情、生成内容。
- 副模型: 设置里的副模型 / API 池，负责后台整理，比如心声、状态栏、事件提取、记忆抽取、总结。
- 下面的“次数”指大模型调用次数，不包括语音转文字、文字转语音、联网搜索、音乐接口、Embedding、Notion、飞书、小红书等非大模型接口。
- 实际次数会因为开关、失败重试、手动重新生成而增加。
- 如果没有配置副模型，很多后台整理不会运行；少数总结功能会改用主模型兜底。

## 最常见的消耗

| 功能 | 一次操作大概调用 |
| --- | --- |
| 私聊发 1 条消息 | 主模型 1 次；如果开了心声/状态栏/记忆等，副模型 0-4 次 |
| 语音通话里说 1 轮 | 主模型 1 次；通话结束整理记忆时副模型 0-1 次 |
| 见面 / 约会里发 1 条消息 | 主模型 1 次；自动总结时副模型 0-1 次 |
| 群聊发 1 条消息 | 主模型 1 次，模型一次性决定群里谁说话、说几条 |
| AI 图片识别 / 食物估算 | 主模型 1 次 |
| 手动点“总结 / 生成 / 重写” | 通常主模型或副模型 1 次，看具体功能 |

## 给用户看的简短版

```text
普通聊天发一条消息，主模型一般调用 1 次。
如果你开启了心声、状态栏、事件提取、记忆抽取等后台功能，发完消息后可能再调用副模型 1-4 次。

见面、约会、TRPG、自习室、小说、存钱罐等生成类功能，通常每点一次“生成/继续/重写”，就是主模型 1 次。

摘星楼主要走副模型，每抽牌、解读或追问一次，大概副模型 1 次。

语音输入、语音朗读、音乐播放、云备份、Notion/飞书同步、小红书真实操作本身不算主/副模型调用；但如果需要 AI 写回复、总结、文案或识别图片，就会额外调用模型。
```
