推荐浏览器

这是按整个 Sully 小手机体验给出的推荐，不只看后台通知。

iPhone / iPad

Chrome / Edge / Firefox / Safari

都可以用。选功能最完整、保存和上传最稳定的那个。Safari 添加到主屏幕不是默认要求。

安卓手机

Chrome

最推荐。不要用微信、QQ、小红书、微博等 App 内置浏览器。

Windows

Chrome / Edge

两个都可以。系统通知和浏览器网站通知都要打开。

macOS

Chrome / Safari

两个都可以。通知样式建议选「横幅」或「提醒」。

怎么设置

按自己的设备看这一栏就够了。

iPhone / iPad

  1. 用 Chrome、Edge、Firefox 或 Safari 打开 Sully。
  2. 选择功能最完整、最少出错的浏览器作为日常入口。
  3. 进入 Sully 设置，打开「系统通知」和「后台回复通知」。
  4. 浏览器或系统弹出通知权限时，点「允许」。
  5. 系统设置 → 通知 → 找到对应浏览器或 Sully → 允许通知。
  6. 关闭专注模式、勿扰模式、低电量模式后再测试。
  7. Safari 添加到主屏幕只作为通知/PWA 尝试；如果功能不完整，就不要用它当日常入口。

安卓手机

  1. 用 Chrome 打开 Sully。
  2. 进入 Sully 设置，打开「系统通知」和「后台回复通知」。
  3. Chrome 弹出通知权限时，点「允许」。
  4. Chrome 地址栏左侧网站设置 → 通知 → 允许。
  5. 系统设置 → 应用 → Chrome → 通知 → 允许。
  6. 系统设置 → 应用 → Chrome → 电池 → 不限制 / 允许后台运行。
  7. 不要把 Chrome 从最近任务里划掉；能锁定后台的话可以锁定。

Windows

  1. 用 Chrome 或 Edge 打开 Sully。
  2. 进入 Sully 设置，打开「系统通知」和「后台回复通知」。
  3. 浏览器弹出通知权限时，点「允许」。
  4. Windows 设置 → 系统 → 通知 → 打开 Chrome / Edge 通知。
  5. 关闭「请勿打扰 / 专注助手」。
  6. 可以最小化浏览器，但不要关闭标签页。

macOS

  1. 用 Chrome 或 Safari 打开 Sully。
  2. 进入 Sully 设置，打开「系统通知」和「后台回复通知」。
  3. 浏览器弹出通知权限时，点「允许」。
  4. 系统设置 → 通知 → Chrome / Safari / Sully → 允许通知。
  5. 通知样式建议选择「横幅」或「提醒」。
  6. 关闭专注模式后再测试。

安卓额外检查

如果安卓收不到通知，优先检查这些。

小米 / Redmi / POCO

Chrome 通知允许；悬浮通知、锁屏通知允许；省电策略改成「无限制」；最近任务锁定 Chrome。

华为 / 荣耀

应用启动管理里把 Chrome 改为手动管理，允许自启动、关联启动、后台活动。

OPPO / realme / 一加

Chrome 通知允许横幅和锁屏；电池里允许 Chrome 后台运行；最近任务锁定 Chrome。

vivo / iQOO / 三星

允许 Chrome 通知；允许后台高耗电或加入不休眠应用；关闭过强省电模式。

能力边界

这部分说清楚能做到什么、不能保证什么。

普通聊天短后台

发完消息后短时间切到后台，回复完成后会尝试进入系统通知栏。

后台主动消息

状态栏、自主代理这类主动运行更挑系统。安卓 Chrome 更适合测试；iOS 整体更容易冻结。

不能保证

关闭网页、清后台、长时间锁屏、极限省电、无痕模式、内置浏览器，都可能收不到。

测试方法

  1. 打开 Sully，进入普通聊天。
  2. 发送：「等下你回复我一段话。」
  3. 立刻切到桌面或打开别的 App。
  4. 等待 1-2 分钟。
  5. 下拉系统通知栏，看是否出现角色回复。

结果判断

  • 通知栏有消息：设置成功。
  • 聊天里有回复但没通知：检查通知权限和横幅权限。
  • 聊天里也没回复：浏览器可能被系统冻结或清后台。
  • 没有顶部横幅但通知中心有消息：这是系统展示策略。
一句话版

iPhone 上 Chrome、Edge、Firefox、Safari 都可以，选功能最完整的；安卓推荐 Chrome；Windows 推荐 Chrome / Edge；macOS 推荐 Chrome / Safari。打开系统通知和后台回复通知，不要关闭网页，不要清后台。
